# Security Policy

## Supported Versions

This project is currently maintained on the `main` branch.

If you discover a security issue, please assume that only the latest code on `main` is supported for fixes unless explicitly noted otherwise.

## Reporting a Vulnerability

If you believe you have found a security vulnerability in this repository, please report it privately.

Preferred options:

1. Use GitHub's private vulnerability reporting feature for this repository if it is enabled.
2. Contact the maintainer directly before opening a public issue.

Please do not disclose the vulnerability publicly before it has been reviewed and triaged.

## What to Include

To help with investigation, please include as much of the following as possible:

- A clear description of the issue
- Steps to reproduce
- The affected feature, endpoint, or file
- Potential impact
- Any proof of concept, logs, screenshots, or payload samples that are safe to share

## Response Expectations

After a report is received, the maintainer will try to:

- Confirm whether the issue is valid
- Assess impact and severity
- Determine whether a fix is needed
- Prepare a patch or mitigation when appropriate

Response times may vary depending on availability, but reports will be reviewed as soon as possible.

## Scope Notes

The following are generally not considered security vulnerabilities by themselves:

- Missing best-practice headers without a demonstrated exploit path
- Rate limiting suggestions without abuse evidence
- Outdated dependencies without a relevant known impact on this project
- Issues that only affect local development setups with no real security impact

## Disclosure

Please allow reasonable time for verification and remediation before any public disclosure.
